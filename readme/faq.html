The Q&A

Q: I get the error message "This application has failed to start because cygwin1.dll was not found. Re-installing the application may fix this problem."
A: You're missing the cygwin dlls. Please get the latest dll at: http://www.cygwin.com/snapshot. If you're unsure, asking around in our IRC chatroom will get you around too, but always remember - use common sense and search before asking.

Q: My map-server won't load! It appears to be loading things before it dissapeared suddenly! HELP!
A: Use command line to load map-server. It should tell you the error. If you're missing a map, update your kRO Sakray or comment the map from map_athena.conf. If you have an errornous NPC, fix it or comment it off. Anything other than that, feel free to ask around.

Q: My map-server failed to load 'adata.grf'! Where do I find this adata.grf? My map-server won't load without it!
A: The error that caused the map-server to not load is not the adata.grf. adata.grf is NOT a requirement for the map-server to load. The error is probably related to something else.

Q: All 3 of the servers are loaded, but I am still having problems accessing it! What do I do?
A: First, check the IPs in map_athena.conf and char_athena.conf. If those are correct, check the ports to make sure they match. If that's correct too, you probably cannot handle the server load. Lower the monster spawning rate using mob_count in battle_athena.conf and it should be fine.

Q: How do I start Guild Wars/War of Emperium??? HELP!!!
A: Read the GM Command page for full list of commands that GMs can use, including the command for this.

Q: My Ragnarok Online crashed while playing with Freya! What do I do now?
A: Well, if your Ragnarok crashes, it's most probably not anything to do with Freya. Something is wrong with your Ragnarok installation. Try reinstalling or updating.

Q: Is Freya compatible with mySQL? Can I use mySQL as the DB instead of using text files?
A: Yes, Freya is compatible with mySQL. A tutorial on how to setup this is coming soon.

Q: Is Freya compatible with msSQL? Can I use msSQL as the DB instead of using text files?
A: No, Freya is not compatible with msSQL. You cannot use msSQL with Freya.

Q: I found a bug! Where do I report it?
A: Drop the developers a line at the IRC chatroom. Or just post it in the bug report forum. We check them out too. :)

Q: I know alot of C and I'm able to help improve Freya and add new features. How can I join your development team?
A: Try talking to one of the current developers in the Freya channel.

Q: We've encountered a wierd bug. Each time the server reaches 1016 players, the map-server doesn't let anyone on.
A: You must resize FD_SIZE on your system. If you're running on Debian (probably similar for other distribs), this is what you need to do to raise the maximum of connections allowed on your server. Edit /etc/security/limits.conf, read it and add a HARD limit on the user you want, set that limit to 2048+ (by increments of 1024 only). Edit /usr/include/bits/typesizes.h, look for the line "#define __FD_SETSIZE 1024", replace the 1024 by 2048+ (again, only by increments of 1024). Go in your RO folder, make clean && make (sql), and this problem should resolved.

Q: Under Cygwin, when I compile for Native Windows (./configure --with-win32), my map-server stops/crashes before to load maps.
A: You must remove/uninstall zlib in Cygwin:
Run setup (of Cygwin), select zlib package in libs category and click on 'remove'. Then, valid.
After, run configure of Freya (./configure + options), and say 'y' when it proposes to crosscompile zlib library.
Re-run configure of Freya after terminated. Then type 'make'.

Q: I can not compile under FreeBSD.
A: Text from [folx] found in Freya's bug report:
(on FreeBSD 5.3-RELEASE, using Freya-txt: all version)
use ./configure --without-gettext to create the Makefile
edit Makefile:
* change MAKE = gmake into MAKE = make
* remove tool and addons from line: all: title common login char ladmin tool map addons
so it must be: all: title common login char ladmin map
* remove the --no-print-directory in eatch category (common, login, char, ladmin, map), i.e.:
common:
@echo -e " - DIR\tsrc/common"
@$(MAKE) --no-print-directory -C src/common/ libcommon.a
should look like :
common:
@echo -e " - DIR\tsrc/common"
@$(MAKE) -C src/common/ libcommon.a
save modified Makefile :D
now compile your own Freya version: make
I don't know why FreeBSD doesn't compile Tool & Add-ons, but what i know is we don't need it to run Freya ;-)

Q: What is the 'import' file for?
A: Most people don't know the real use of the 'import' files. After you do, you will wonder what you ever did without it.
The main thing it does, is provide a way for you to change your config settings without having to modify the actual files. You can test some settings by this way.

Q: How does the 'import' file work?
A: You place only the settings you have changed in the 'import' files.
Say you want to change your job experience rate from the default (100)to 2x (200). Well then you would place this in your import/battle_conf.txt (folder and name of the 'import' file are mentionned in your 'import' line):
job_exp_rate: 200
You don't need the comment (it's only a comment). You can add it for clarity sake.
Now this new setting takes place over the setting in battle_athena.conf.
How? Simple. When the server reads the 'import' line, it reads the mentioned file and all values inside this file replace previously readed values. The last readed setting of each option is the used setting.
But, if you add another 'import' file with this option or change this option after the 'import' line, your 'import' setting will not be used. The last value is always the value used by the server.
